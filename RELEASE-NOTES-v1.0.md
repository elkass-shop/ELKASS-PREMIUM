# ELKASS v1.0 Production Candidate

Połączono cztery etapy w jedno repo. Dodano zbiorczą, idempotentną migrację Supabase, role panelu, bezpieczny Storage, katalog, CMS, klientów, zamówienia, koszyk i finalizację zamówienia. Zachowano istniejący wygląd panelu i frontu.

To wydanie jest kandydatem produkcyjnym. Uruchomienie płatności online wymaga wyboru operatora, umowy, sekretów serwerowych i webhooka — nie są one umieszczane w repo frontendowym.
